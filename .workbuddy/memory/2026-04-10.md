# 2026-04-10 工作日志

## 商业化讨论 + 用户认证+付费功能开发
- 用户决定以 Web 端为主力，暂不做小程序同步
- 初步尝试了**付费买断模式**（feature-premium分支），但用户意识到 ¥19.9×1000人=2w 的投入产出比不高
- **最终策略转变：先做完全免费版本，通过数据采集积累用户行为数据，探索更高价值的变现方式**
  - 方案 A：免费工具 + 数据变现（行业报告、报价白皮书卖给品牌方/MCN）
  - 方案 B：双边平台（商达撮合，品牌方付费找博主）
  - 方案 C：年费制替代终身买断（¥79/年 vs ¥19.9 一次，3年收入差10倍）
  - 方案 D：增值服务抽佣（对账单/报税辅助等强痛点功能）
- **Git 分支结构**：
  - `main` = 当前工作分支（免费版 + Tracker 数据采集系统）
  - `feature-premium` = 付费登录版（已保存，随时可切回）

## Tracker 数据采集系统（已集成到 main 分支）
- 匿名用户追踪（本地生成随机UID，不涉及个人信息）
- 覆盖全部关键行为：
  - 页面访问 / Tab 切换
  - 商单 CRUD（创建时记录脱敏聚合数据：价格区间、内容形式、行业分类等）
  - 状态变更追踪（from → to）
  - 日历拖拽改期
  - 品牌添加
  - 设备信息（屏幕尺寸、浏览器类型、是否移动端）
  - 会话统计（活跃天数、访问次数）
- 存储机制：localStorage 本地缓存 + 批量上报架构（30秒一批或满20条即发）
- 隐藏管理面板：连续点击5次导航栏标题可打开查看本地统计数据
- 一键导出 JSON 功能（供开发者分析用）
- 所有数据均为脱敏聚合数据（价格分档而非原始值，不含手机号姓名等PII）
